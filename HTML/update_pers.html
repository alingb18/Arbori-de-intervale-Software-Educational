Funcția Update (Arbori de intervale persistenți)
Ca introducere, de fiecare dată când se va actualiza o poziție, se vor schimba toate nodurile de pe lanțul de la rădăcină la frunza respectivă. De aici rezultă că se vor modifica maxim log(N) noduri pentru fiecare versiune a structurii. Acest aspect conferă și complexitatea în memorie a arborilor de intervale persistenți care va fi O(N+Q*log(N)), unde Q este numărul de operații de update. La fel ca în cazul funcției Build, funcția Update funcționează aproape la fel ca în cazul structurii clasice, diferența fiind că de data aceasta returnează adresa nodului creat. Această mențiune implică faptul că adresa de memorie a nodului nu există în momentul apelului funcției, deci nu vom putea reține acest parametru ca în cazul arborilor de intervale clasici. Așadar, funcția va avea 4 parametri: {st, dr} care definesc intervalul de acoperire al nodului actual și {poz, val} care definesc actualizarea. Pentru simplitate, la crearea nodului am folosit un constructor definit în struct. Complexitate: O(log(N))
Cod 

        
node* update(node* nod, int st, int dr, int poz, int val)
{
    if(st==dr) return new node(val);
    int mij=(st+dr)/2;
    if(poz<=mij) return new node(update(nod->l, st, mij, poz, val), nod->r);
    else return new node(nod->l, update(nod->r, mij+1, dr, poz, val));
}
